
# Python网络数据爬虫与可视化分析系统 - 开发需求文档

## 1. 项目概述

### 1.1 项目背景
开发一套基于Python技术栈的轻量级网络数据爬虫与Web可视化分析系统，实现数据采集、清洗、存储、可视化的全流程服务，让用户无需编写代码即可快速完成互联网数据的抓取与分析。

### 1.2 项目目标
- 实现基础网页数据的定向抓取
- 提供简洁易用的Web管理界面
- 实现数据的结构化存储与查询
- 提供基础的数据可视化功能
- 确保系统稳定可靠，易于维护

## 5. 数据库设计

### 5.1 用户表 (users)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| username | VARCHAR(50) | 用户名，唯一 |
| password | VARCHAR(255) | 密码（哈希存储） |
| created_at | DATETIME | 创建时间 |
| PRIMARY KEY | (id) |  |
| UNIQUE KEY | (username) |  |

### 5.2 爬虫任务表 (tasks)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 关联用户ID |
| name | VARCHAR(100) | 任务名称 |
| url | VARCHAR(500) | 目标URL |
| selector_config | JSON | 提取规则配置 |
| status | VARCHAR(20) | 任务状态（pending/running/completed/failed） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| PRIMARY KEY | (id) |  |
| KEY | (user_id) |  |
| KEY | (status) |  |

### 5.3 抓取数据表 (data)
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键，自增 |

| content | TEXT | 内容 |
| url | VARCHAR(500) | 源URL |
| extra | JSON | 扩展字段 |
| created_at | DATETIME | 抓取时间 |
| PRIMARY KEY | (id) |  |
| KEY | (task_id) |  |
| KEY | (created_at) |  |

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 网站反爬机制 | 中 | 优先选择反爬较弱的公开网站作为测试目标 |
| 动态页面抓取 | 中 | 初期只支持静态页面，后期可考虑添加Playwright |
| 开发进度延误 | 中 | 优先实现P0功能，P1功能可延后 |

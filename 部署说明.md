# 部署说明

## 1. 环境准备

- Python 3.9+
- Node.js 18+
- MySQL 8.0+

## 2. 配置 MySQL

1. 启动本地 MySQL 服务
2. 创建数据库账号，或确认已有账号可连接
3. 在 `backend/.env` 中配置数据库连接信息

示例：

```env
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=crawler_system
```

## 3. 初始化后端

```bash
python3 -m venv .venv
./.venv/bin/pip install -r backend/requirements.txt
./.venv/bin/playwright install chromium
./.venv/bin/python create_db.py
cd backend
../.venv/bin/python app.py
```

后端默认地址：`http://localhost:5000`

## 4. 初始化前端

```bash
cd frontend
npm install
npm run dev
```

前端默认地址：`http://localhost:3001`

## 5. 验收建议流程

1. 注册并登录系统
2. 创建一个任务，配置 URL、请求方式和提取字段
3. 启动任务，观察进度和日志
4. 在任务详情页确认抓取结果
5. 在数据管理页执行搜索、删除和导出
6. 在可视化页确认统计卡片、饼图、柱状图和折线图

## 6. 常见问题

### MySQL 连接失败

- 确认 MySQL 服务已启动
- 确认 `DB_HOST / DB_PORT / DB_USER / DB_PASSWORD / DB_NAME` 配置正确
- 确认本机防火墙没有拦截 3306 端口

### 前端无法请求后端

- 确认 Flask 已启动在 `5000` 端口
- 确认前端通过 `npm run dev` 启动在 `3001` 端口
- 确认浏览器没有缓存旧页面

### 动态网页抓不到内容

- 确认已经执行过 `./.venv/bin/playwright install chromium`
- 动态网页优先选择“动态渲染”加载方式
- 如果页面内容出现较慢，可以填写“等待选择器”或增加“额外等待(毫秒)”
